VDT-AIOps — Kiến trúc & Hạ tầng v1.3 · cập nhật 2026-06-28

Tài liệu sống mô tả toàn bộ hệ thống: nửa bị giám sát (simulation stack) và nửa đi giám sát (AIOps engine + observability + TUI). Pipeline đầy đủ detect → correlate → diagnose (AI) → persist → TUI đã chạy end-to-end.

DONE đã hoàn thiện WIP đang làm TODO chưa làm PLANNED thiết kế / dự kiến

2. Hạ tầng — 4 khối

Khối 1 — Simulation stack DONE

8 container giả lập web stack điển hình — đối tượng bị giám sát. File aiops/infra/docker-compose.yml.

Request path: traffic-gen bắn HTTP liên tục → nginx:80 → node-api:3000 → redis (cache-aside), miss thì postgres.

Khối 2 — Observability + Engine (container) DONE

File aiops/infra/docker-compose.backend.yml (join network external: true):

  • prometheus (:9090) — scrape node-api trực tiếp + nginx/postgres/redis qua exporter.
  • postgres-aiops (:5433→5432) — state store: service_logs, alerts, incidents. Flyway tự dựng schema lúc khởi động.
  • docker-proxy (socat) — bắc cầu /var/run/docker.sock → TCP 2375 nội bộ (xem mục 3).
  • aiops-engine (Spring Boot, :8088) — bộ não always-on: collector · scraper · detector · correlate · AI agent · WebSocket server.

Khối 3 — TUI (React Ink) DONE

Chạy ở terminal host (npm start, không đóng container). Nối engine qua WebSocket ws://localhost:8088/ws/incidents. Hiển thị snapshot incident active + sự kiện điều tra live + chat hỏi-đáp; thao tác ack/resolve. Chi tiết mục 8.

3. Truy cập Docker API — socat proxy DONE

Vì sao KHÔNG dùng unix:///var/run/docker.sock trực tiếp: đã thử & chứng minh — OS-level socket OK, nhưng transport docker-java-transport-httpclient5 3.3.6 khi nhận DOCKER_HOST=unix://… lại đi TCP tới host giả localhost:2375 (không làm unix domain socket) → Connection refused. Ngoài ra Dockerfile chạy USER aiops (non-root) → permission denied trên socket.

Giải pháp: container docker-proxy (image alpine/socat, chạy root, mount socket) bắc cầu UNIX-CONNECT:/var/run/docker.sock ↔ TCP-LISTEN:2375. Engine nối TCP (docker-java chạy TCP ổn định):

Chạy ở đâuDOCKER_HOSTGhi chú
Engine trong container (handoff)tcp://docker-proxy:2375resolve qua DNS service nội bộ Docker
Engine local bằng mvn (dev)tcp://localhost:2375 (default)proxy publish 127.0.0.1:2375 → khỏi bật toggle 2375

Lợi ích handoff: giám khảo không phải bật "Expose daemon on tcp://localhost:2375 without TLS" trong Docker Desktop — compose up là chạy. TCP chỉ mở trong mạng nội bộ (và 127.0.0.1 cho dev).

Quy tắc cấu hình: default trong application.yaml = giá trị cho local (localhost); container đè bằng env qua compose (12-Factor). Cùng nguyên tắc cho DB (localhost:5433 ↔ postgres-aiops:5432) và Prometheus (localhost:9090 ↔ prometheus:9090).

4. AIOps engine — 2 data plane DONE

Log plane

LogCollector: mỗi container 1 virtual thread stream docker logs → buffer → flush vào service_logs mỗi 2s. LogTemplateExtractor gom log theo template để giảm nhiễu. parseLevel: ERROR/WARN/INFO (nginx 4xx-5xx → ERROR).

Metric plane

MetricCollector mỗi 5s: docker stats (CPU/mem) + PromQL qua PrometheusClient theo profile từng service (MetricProfiles): vd node-api errorRate (lỗi/giây), latencyMs (P99 histogram), redis memUsedPercent, postgres connUtilPercent, isUp.

5. Alert pipeline — phát hiện → gom nhóm → debounce dispatch DONE

Hai vòng lặp @Scheduled độc lập, giao tiếp qua DB:

VÒNG A (5s)  AnomalyDetector.scan() → AlertManager.handle()  → GHI alert vào DB
                                                                    │ (DB = điểm gặp)
VÒNG B (5s)  CorrelationTick ← ĐỌC alert (active=true, status=ACTIVE) ┘
             → correlate() (DSU) → AlertGroup → debounce chín? → dispatch @Async → AGENT
  • AnomalyDetector: duyệt rule aiops.anomaly.rules, so breached(value). AlertManager: tạo + dedup; resolve CHỈ khi đo được HEALTHY (không false-resolve lúc tín hiệu null/service chết).
  • Trạng thái 2 trục active (liveness, Detector ghi) × status ACTIVE→INVESTIGATING→DIAGNOSED (Agent ghi). Correlate lấy giao ô active=true ∧ ACTIVE — chống gửi lặp + tự dập blip nhất thời.
  • AlertCorrelator (DSU): gom alert cùng/kề service ≤ 5 phút thành AlertGroup; rootCandidate = service bị phụ thuộc nhiều nhất trong cụm. Gom THÔ — agent sẽ tách 1 group → nhiều incident.
  • CorrelationTick (stateless recompute): dispatch khi now−updatedAt ≥ quietWindow(30s) HOẶC now−createdAt ≥ maxWait(120s); mark INVESTIGATING để dedup.

6. AI Agent — vòng lặp Query 4 pha DONE

Query.investigate(bundle, investigationId) là agentic loop tự viết (KHÔNG để Spring AI tự chạy tool — internalToolExecutionEnabled=false), lặp đến điều kiện dừng:

1
Assembly Context — ghép ContextBundle (log+metric của service lỗi & lân cận theo ServiceGraph) + schema JSON của List<IncidentReport> (qua BeanOutputConverter) vào prompt. AutoCompact tóm tắt nếu vượt token budget.
2
Model Call — chatModel.call(prompt) với ToolCallingChatOptions (đăng ký diagnosticTools). Investigation dùng call blocking (output JSON cấu trúc); chat mode mới dùng stream.
3
Tool Execution — needsFollowUp = response.hasToolCalls(). Nếu có: publish AgentTurnEvent (TUI hiện tool đang gọi) → ToolCallingManager.executeToolCalls() → nạp kết quả vào history → quay lại pha 2.
4
Stop / Force-final — lặp tới khi hết tool-call hoặc chạm maxTurns(8). Nếu hết lượt mà còn đòi tool → forceFinalAnswer() ép xuất JSON cuối. Parse qua JsonExtractor → List<IncidentReport>.

Read tools (@Tool) DONE

Đăng ký tường minh trong AgentConfig.ToolCallbacks.from(...) (không auto-discover); dùng chung cho cả investigate & chat:

getServiceLogs · getServiceMetrics · getServiceDependencies · queryMetrics · getActiveAlerts · getIncidentHistory · inspectContainer · listActiveIncidents · getIncident · getRecentIncidents. Output cuối ghi qua SaveIncidentReport.

ServiceGraph — auto-discover từ Docker DONE

Đồ thị phụ thuộc có hướng dựng từ compose labels + env (DATABASE_URL, REDIS_URL). Downstream = nơi tìm root cause; Upstream = blast radius. Node *-exporter loại khỏi traversal.

8. TUI (React Ink) DONE

  • Layout: Header (logo + DUCKOMPOSE, viền chạy sáng) / Viewport cuộn theo dòng / InputBar / StatusBar. Theme Panda (nền tối, phối xanh-vàng).
  • Hai chế độ: CHAT (gõ hỏi, agent tự tra qua tool) ↔ SELECT (Tab; ↑↓ chọn incident, a/r/e ack/resolve/expand).
  • Hiển thị điều tra live: alert group → danh sách tool (tên + tham số + giờ, tự thu gọn khi xong, click mở) → incident (severity color-code, mở rộng xem summary/findings/hypotheses/remediation/evidence).
  • Chuột: lăn cuộn · kéo scrollbar · click dòng-icon để mở/thu · click nút [a][r][e] · click tiêu đề mục để copy (OSC 52). Bôi đen text: Shift+kéo (native terminal).
  • Stream mượt: token gom & nhả kiểu "đánh máy"; thoát bằng Ctrl+Q (Ctrl+C tắt để copy an toàn).

Backend (Spring AI 1.1.x) lọc emoji ở prompt; TUI lọc thêm ở tầng render (giữ ký hiệu văn bản). Conversation store in-memory (TTL 30') — nhớ trong phiên là đủ cho SRE, không persist lịch sử.

10. Quyết định kiến trúc (Decision log)

Log lưu PostgreSQL (không Elasticsearch). Một nguồn dữ liệu (postgres-aiops) cho log + state. ES trong CLAUDE.md gác lại.
Tách CLIENT–SERVER + WebSocket. Bỏ Spring Shell; TUI React Ink (host, npm) ⇄ Java engine always-on. Transport WebSocket (raw TextWebSocketHandler + JSON discriminated-union theo type, không STOMP). Bật lại Tomcat (đảo ràng buộc no-Tomcat/web-type-none trong CLAUDE.md).
Alert 2 trục (active × status) + Correlation gom THÔ. Cần cả 2 trục để phân biệt "đã chẩn mà vẫn cháy" vs "đã chẩn & tắt". DSU gom rộng (chấp nhận over-merge do victim-bridge), agent là trọng tài tách 1 AlertGroup → nhiều Incident.
Agent loop tự viết (LOCKED). internalToolExecutionEnabled=false + tự lặp; needsFollowUp = hasToolCalls; structured output List<IncidentReport> qua BeanOutputConverter; maxTurns guard + forceFinalAnswer. Investigate = blocking call; chat = stream (MessageAggregator giữ tool qua aggregation).
2026-06-28 — Truy cập Docker qua socat TCP proxy (không unix://). docker-java-transport-httpclient5 không làm được unix domain socket (đi TCP tới placeholder → refused); container non-root còn bị permission denied. Chốt: sidecar alpine/socat bắc cầu socket → TCP nội bộ; engine nối tcp://docker-proxy:2375. Giám khảo khỏi bật toggle 2375 → zero-config khi vận chuyển code.
Bài học Flyway: KHÔNG sửa migration đã apply. Đổi checksum → Validate failed. Cần đổi thì thêm V(n+1) idempotent (ADD COLUMN IF NOT EXISTS).

11. Chạy hệ thống (handoff)

Yêu cầu: Docker Desktop (đang chạy) · Node.js 18+ (cho TUI). Không cần cài Java/Maven (engine build trong Docker). Không cần bật daemon TCP 2375.

# 1) Điền khóa: tạo aiops/infra/.env từ .env.example -> đặt ANTHROPIC_API_KEY (+ 2 password)
# 2) Dựng hạ tầng + engine (sim TRƯỚC để tạo network):
cd aiops/infra
docker compose up -d                                        # simulation stack
docker compose -f docker-compose.backend.yml up -d --build  # engine + proxy + prometheus + state-db
# 3) Chạy TUI:
cd ../tui && npm install && npm start
# 4) Tạo sự cố demo (PowerShell):
./scripts/simulate-redis-oom.ps1        # hoặc simulate-minor-latency.ps1 (P4)
2 file .env riêng: infra/.env (cho engine trong Docker) ‖ app/.env (cho chạy mvn local). Khoá sai/placeholder → Claude trả 401 invalid x-api-key → TUI hiện "Sorry, I can't handle this question". Cả hai đều gitignore (*.env).
